What this is

Pick-and-place on a UR7e arm with a RealSense RGB-D camera, inverse kinematics, and a joint-space MPC that handles the motion. Camera sees objects, IK figures out joint targets, MPC gets there while respecting joint limits, velocity bounds, and collision margins. The whole loop runs online so the robot can react when things shift around.

How it works

The RealSense gives us RGB-D for object detection and 6-DOF pose estimation. Targets update live as new objects appear. Depth data is noisy and there's partial occlusion, so the perception has to be robust to that.

On the control side, a joint-space MPC optimizes over a finite horizon each timestep with hard constraints on positions, velocities, and collision clearance. If the target moves mid-trajectory, the controller replans without restarting from scratch. Getting that replan to be fast enough was one of the trickier parts. Early versions would stutter when the target jumped.

ROS 2 + MoveIt ties it all together: camera feed into IK into MPC into motor commands. Each piece is independently testable, which made debugging a lot less painful when things inevitably broke at integration time.

Results

  • Collision-free pick-and-place in cluttered setups
  • Smooth trajectories within all joint constraints
  • Replans in under a second when targets shift

Stack
