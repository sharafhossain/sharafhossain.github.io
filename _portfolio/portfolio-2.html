What this is

A pick-and-place system on a UR7e arm. The camera sees the objects (RealSense RGB-D), inverse kinematics figures out where the joints need to go, and a joint-space MPC gets them there while respecting joint limits, velocity bounds, and collision margins. The whole loop runs online so the robot can react when things move.

This runs on actual hardware in a cluttered environment. Not a simulation screenshot.

How it works

Perception

  • RGB-D object detection and 6-DOF pose estimation with RealSense
  • Targets update online as new objects come into view
  • Handles partial occlusion and noisy depth data

Control

  • Joint-space MPC optimizes over a finite horizon each timestep
  • Hard constraints on joint positions, velocities, and collision clearance
  • If the target moves mid-trajectory, the controller replans without starting over

Integration

  • ROS 2 + MoveIt ties it together: camera feed → IK → MPC → motor commands
  • Each piece is modular and independently testable
  • Validated on the physical UR7e arm

Results

  • Collision-free pick-and-place in cluttered setups
  • Smooth trajectories that stay within all joint constraints
  • Replans in under a second when targets shift

Stack

Python, C++, ROS 2, MoveIt, ACADOS, RealSense, UR7e
